# AI-TestHub 智能测试平台 PRD

## 1. 项目概述

**AI-TestHub** 是一款面向企业的智能化接口自动化测试平台，通过 AI 技术辅助测试用例生成和测试结果分析，降低测试门槛，提升测试效率。

**当前版本：** v1.0  
**产品角色：** 📋 产品经理  
**更新时间：** 2026-03-12

---

## 2. 背景与目标

### 2.1 背景

随着微服务架构的普及，企业内部的接口数量急剧增长。传统的手工测试方式面临以下挑战：

- **用例维护成本高**：接口变更后需要手动更新大量测试用例
- **覆盖率难以提升**：人工难以覆盖所有边界场景
- **测试效率低下**：重复性测试占用大量人力
- **AI 能力缺失**：缺少智能化的测试辅助能力

### 2.2 目标

v1.0 阶段目标：

- 建立基础的项目管理能力
- 实现接口管理与测试用例的关联
- 集成 AI 能力辅助测试用例生成
- 提供基础的测试执行与报告能力

## 3. 功能模块

### 3.1 项目管理模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 项目列表 | 查看所有测试项目 | P0 |
| 创建项目 | 新建测试项目 | P0 |
| 编辑项目 | 修改项目名称、描述 | P0 |
| 删除项目 | 删除项目及其关联数据 | P1 |

**数据模型：**
```python
class Project:
    id: int
    name: str          # 项目名称
    description: str   # 项目描述
    created_at: datetime
    updated_at: datetime
```

### 3.2 接口管理模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 接口列表 | 查看项目下所有接口 | P0 |
| 创建接口 | 新建接口定义 | P0 |
| 编辑接口 | 修改接口信息 | P0 |
| 删除接口 | 删除接口 | P0 |
| 接口调试 | 快速测试接口 | P1 |

**数据模型：**
```python
class Interface:
    id: int
    project_id: int    # 所属项目
    name: str          # 接口名称
    method: str        # GET/POST/PUT/DELETE
    url: str           # 请求地址
    description: str   # 描述
    headers: dict      # 请求头
    params: dict       # URL参数
    body: dict         # 请求体
    body_type: str     # json/form-data/raw
```

### 3.3 测试用例模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用例列表 | 查看项目下所有用例 | P0 |
| 创建用例 | 新建测试用例 | P0 |
| 编辑用例 | 修改用例配置 | P0 |
| 删除用例 | 删除测试用例 | P0 |
| 执行用例 | 批量/单次执行用例 | P0 |
| AI 生成 | AI 自动生成测试用例 | P1 |

**数据模型：**
```python
class TestCase:
    id: int
    project_id: int
    interface_id: int  # 关联接口
    name: str
    description: str
    request_config: dict  # 请求配置
    assertions: dict       # 断言配置
```

### 3.4 测试执行与报告

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 执行测试 | 执行单个或批量测试用例 | P0 |
| 执行记录 | 查看历史执行结果 | P0 |
| 测试报告 | 查看测试结果统计 | P0 |
| AI 总结 | AI 分析测试结果 | P1 |

**数据模型：**
```python
class TestResult:
    id: int
    case_id: int
    status: str        # success/fail/error
    response: dict     # 响应数据
    error_message: str # 错误信息
    duration_ms: int   # 执行耗时
    executed_at: datetime
```

### 3.5 插件管理（扩展性）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 插件列表 | 查看已安装插件 | P1 |
| 启用/禁用 | 控制插件开关 | P1 |
| 插件配置 | 配置插件参数 | P2 |

## 4. 技术架构

### 4.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React 18 + TypeScript + Ant Design 5 + Vite |
| 后端 | Python FastAPI + SQLAlchemy |
| 数据库 | SQLite（默认，可扩展 MySQL/PostgreSQL）|
| AI 集成 | OpenAI API（预留接口）|

### 4.2 API 设计规范

```
/api/projects          # 项目管理
/api/interfaces        # 接口管理
/api/cases             # 测试用例
/api/results           # 测试结果
/api/ai                # AI 功能
/api/plugins           # 插件管理
```

## 5. 用户流程

### 5.1 基础使用流程

```
1. 创建项目
   ↓
2. 添加接口（手动或导入）
   ↓
3. 创建测试用例（手动或 AI 生成）
   ↓
4. 执行测试
   ↓
5. 查看报告
```

### 5.2 AI 辅助流程

```
1. 输入需求描述
   ↓
2. AI 生成测试用例
   ↓
3. 人工审核/调整
   ↓
4. 执行测试
```
