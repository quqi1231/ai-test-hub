## 二、数据表结构

### 2.1 现有表（v1.0）

#### 2.1.1 users - 用户表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键 |
| username | VARCHAR(50) | ✅ | 用户名（唯一）|
| email | VARCHAR(100) | ✅ | 邮箱（唯一）|
| hashed_password | VARCHAR(255) | ✅ | 加密密码 |
| is_active | BOOLEAN | ✅ | 是否激活（默认true）|
| created_at | DATETIME | ✅ | 创建时间 |

#### 2.1.2 projects - 项目表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键 |
| name | VARCHAR(100) | ✅ | 项目名称 |
| description | TEXT | ❌ | 项目描述 |
| created_at | DATETIME | ✅ | 创建时间 |
| updated_at | DATETIME | ✅ | 更新时间 |

#### 2.1.3 interfaces - 接口表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键 |
| project_id | INTEGER | ✅ | 外键 → projects.id |
| name | VARCHAR(100) | ✅ | 接口名称 |
| method | VARCHAR(10) | ✅ | 请求方法（GET/POST/PUT/DELETE）|
| url | VARCHAR(500) | ✅ | 请求地址 |
| description | TEXT | ❌ | 接口描述 |
| headers | JSON | ❌ | 请求头 |
| params | JSON | ❌ | URL参数 |
| body | JSON | ❌ | 请求体 |
| body_type | VARCHAR(20) | ✅ | 请求体类型（json/form-data/x-www-form-urlencoded/raw）|
| content_type | VARCHAR(50) | ❌ | Content-Type |
| created_at | DATETIME | ✅ | 创建时间 |
| updated_at | DATETIME | ✅ | 更新时间 |

#### 2.1.4 test_cases - 测试用例表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键 |
| project_id | INTEGER | ✅ | 外键 → projects.id |
| interface_id | INTEGER | ❌ | 外键 → interfaces.id |
| name | VARCHAR(100) | ✅ | 用例名称 |
| description | TEXT | ❌ | 用例描述 |
| request_config | JSON | ❌ | 请求配置 |
| assertions | JSON | ❌ | 断言配置 |
| created_at | DATETIME | ✅ | 创建时间 |
| updated_at | DATETIME | ✅ | 更新时间 |

#### 2.1.5 test_results - 测试结果表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键 |
| case_id | INTEGER | ✅ | 外键 → test_cases.id |
| status | VARCHAR(20) | ✅ | 状态（success/fail/error）|
| response | JSON | ❌ | 响应数据 |
| error_message | TEXT | ❌ | 错误信息 |
| duration_ms | INTEGER | ❌ | 执行耗时（毫秒）|
| executed_at | DATETIME | ✅ | 执行时间 |

#### 2.1.6 plugins - 插件表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | ✅ | 主键 |
| name | VARCHAR(100) | ✅ | 插件名称 |
| description | TEXT | ❌ | 插件描述 |

```json
{

  "get_user": {

}
```

## 五、变量提取机制

### 5.1 变量语法

```markdown
# 环境变量
{{base_url}}
{{token}}

# 响应提取
{{login.response.body.data.token}}
{{login.response.headers.set-cookie}}

# 全局变量
{{global.user_id}}
```

### 5.2 提取规则配置

```json
{
  "extractions": [
    {
      "source": "login",
      "path": "body.data.token",
      "variable": "auth_token"
    },
    {
      "source": "login", 
      "path": "body.data.user_id",
      "variable": "user_id"
    }
  ]
}
```

## 六、执行流程

### 6.1 接口链执行流程

```
1. 加载环境配置
   ├── 获取 base_url
   └── 获取环境变量（token等）

2. 初始化全局变量
   ├── 加载全局变量
   └── 加载上次保存的变量

3. 依次执行接口
   ├── 替换 URL 中的变量
   ├── 替换请求头中的变量
   ├── 替换请求体中的变量
   ├── 发送请求
   ├── 提取响应中的变量
   └── 保存到变量池

4. 保存执行结果
   ├── 记录每个接口的响应
   ├── 记录执行状态
   └── 保存提取的变量供后续使用

5. 生成报告
   ├── 统计通过/失败数
   ├── 记录耗时
   └── 保存到数据库
```
